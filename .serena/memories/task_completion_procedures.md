# slash-commands タスク完了時の手順

## 完了前チェックリスト
- [ ] ワークフローの目的が達成されている
- [ ] 生成されたドキュメントの品質が基準を満たしている
- [ ] 依存関係が適切に処理されている
- [ ] エスカレーションが必要な場合は対応済み

## 文書化手順
1. **変更内容の確認**
   - `git status`で変更ファイルを確認
   - `git diff`で変更内容をレビュー

2. **コミットメッセージ作成**
   - `/dev-6-commit-message`を実行して適切なコミットメッセージを生成
   - Story IDまたは課題番号を含める

3. **コミット実行**
   - `git add .`で変更をステージング
   - `git commit -m "message"`でコミット
   - `git push`でリモートへプッシュ

4. **関連ファイル更新**
   - README.mdに新規ワークフローを追記（必要な場合）
   - 変更履歴を更新（必要な場合）
   - 関連ドキュメントのリンクを確認

## 品質保証手順
1. **セルフレビュー**
   - 作成したワークフローの各ステップを確認
   - 命名規約とフォーマットが遵守されているか確認
   - セキュリティと権限に関する記述が適切か確認

2. **テスト実施**
   - ワークフローの実行可能性を確認
   - 参照ファイルのパスが正しいか確認
   - 依存ワークフローが存在するか確認

3. **フィードバック収集**
   - チームメンバーによるレビューを依頼
   - 改善提案を記録し、次回の改善に反映

## クリーンアップ
- 一時ファイルを削除
- 不要なコメントやデバッグ情報を除去
- ワークスペースを整理整頓

## 報告
- 完了したタスクの概要を報告
- 次のステップまたは関連タスクを提案
- 学習した教訓や改善点を記録
